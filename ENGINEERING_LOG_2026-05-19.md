# 工程日志 2026-05-19

## Auto Memory 跨对话长期记忆

新增文件化长期记忆与 FTS5 检索引擎：

- 设计 `memory/` 长期记忆目录，包含 `MEMORY.md` 索引和多主题 `.md` 记忆文件。
- 每轮 Agent 会话结束后自动写入长期记忆，记忆文件首行生成 150 字以内摘要。
- 接入 LLM 摘要生成，失败时降级为本地摘要，保证记忆可持续写入。
- 使用 SQLite FTS5 建立全文检索索引，支持新会话按当前问题检索数周前的相关对话细节。
- 实现碎片合并机制，将高度重叠的记忆文件合并为概括性文件，并删除旧文件与旧索引行。
- 实现索引瘦身，`MEMORY.md` 总行数控制在 200 行以内。
- Agent 生成回答前会从 FTS5 检索相关长期记忆，并按需注入上下文，实现跨对话记忆。
